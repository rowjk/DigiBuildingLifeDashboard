# Survival Dashboard

本專案是一個為企業內部設計的即時生活資訊儀表板（Dashboard），整合了天氣預報、新聞頭條、YouBike 2.0 即時站點、公車到站資訊、即時路況監控、周邊地圖與美食推薦，並包含後台 CRUD 管理系統。

## 技術與架構
* **網頁前端與互動**：Streamlit (純 Python 響應式網頁框架)
* **資料儲存**：SQLite (本機內嵌型資料庫 `dashboard.db`)
* **資料庫 ORM**：SQLAlchemy
* **外部 API 與套件**：
  * 中央氣象署 (CWA) 天氣 API
  * 台北市與國道 VD 路況 API / 公車到站動態 API / YouBike 2.0 API
  * Google Places API (美食搜尋)
  * `feedparser` (Google 新聞 RSS 串接)
* **資料介接與防禦快取**：
  * 使用 Streamlit 全域快取裝飾器限制第三方 API 連線頻率，防止來源 IP 被限制或鎖定。
  * 針對公車靜態站點資料，採用 `@st.cache_resource` 進行快取，防止重新整理時重複進行 Pickle 深拷貝反序列化的負擔。
  * 導入經緯度 Bounding Box 粗篩過濾，減少 99.5% Haversine 三角函數運算，提昇效能。
  * 容錯降級機制：當 API 斷網、請求超時或連線失敗時，自動顯示最後一次成功快取的暫存資料或高擬真 Mock 資料。

## 安裝與執行步驟

### 1. 系統需求
* Python 3.10 或以上版本。

### 2. 下載專案與安裝套件
請於終端機（Command Prompt 或 PowerShell）中執行以下指令安裝所需依賴套件：
```bash
pip install -r requirements.txt
```

### 3. 配置環境變數 (`.env`)
本專案嚴格限制將 API 金鑰寫死於程式碼中。請在專案根目錄下建立 `.env` 檔案（已為您自動產生），並配置中央氣象署授權碼：
```env
CWA_API_KEY=您的授權碼
# 若有 Google Places API 金鑰，可於此配置：
# GOOGLE_PLACES_API_KEY=您的Google金鑰
```
*(註：請在專案根目錄下的 .env 檔案中寫入您的個人授權碼，格式如：CWA-XXXXXXXXXXXXXXXX)*

### 4. 初始化資料庫與預建餐廳資料
執行 `database.py` 以建立 `dashboard.db` 資料庫，並預建 **20 筆內湖石潭路周邊精選餐廳**、初始公告與預設管理員帳密：
```bash
python database.py
```
* **預設管理員帳號**：`admin`
* **預設管理員密碼**：`admin` (雜湊 SHA256 加密儲存)

### 5. 啟動 Dashboard 網站
使用 Streamlit 啟動網頁服務：
```bash
streamlit run app.py
```
或直接雙擊 `啟動Survival_Dashboard.bat` 啟動服務。啟動後，瀏覽器將自動開啟並導向本機網址（通常為 `http://localhost:8501`）。

## 功能展示說明

### 📌 即時生活資訊

1. **即時天氣監控**：
   * 定位台北市內湖區，展示即時溫度、體感溫度、降雨機率與天氣現象。
   * **水平平行排版**：左側為超大目前溫度字體；右側以一根左邊框線隔開，並排顯示天氣現象、體感溫度與降雨機率。
   * **Unicode 圖示化**：將天氣現象動態解析並替換為單色 Unicode 圖示（如 `⛈️`、`🌧️`、`☁️`、`☀️` 等）。
   * **溫度與體感溫度分色**：溫度大於等於 30°C 顯示為紅色 (`#CC0000`)；介於 20°C 至 29°C 顯示為綠色 (`#00AA00`)；低於 20°C 顯示為藍色 (`#0000AA`)。
   * **降雨機率分色**：降雨機率小於等於 30% 顯示為藍色 (`#0000AA`)；大於 30% 顯示為紅色 (`#CC0000`)。

2. **新聞頭條**：
   * 即時串接 Google 台灣區頭條新聞 RSS，將最新鮮的新聞**依發布時間由新到舊降序排列**，最多展示前 25 則。
   * **卡片式排版與溢出忽略**：新聞卡片限制為固定的 `height: 105px; overflow: hidden;` 大小，新聞標題最多顯示兩行，並在滑鼠懸浮時呈現 translate 向上漂浮微動畫。
   * **點選卡片另開分頁**：卡片包裹於 `_blank` 連結中，點擊即可於新分頁查看完整新聞。

3. **YouBike站點**：
   * 簡化標題為 **`YouBike站點`**，不論 API 正常與否，系統皆會根據「平台中心定位設定」的當前經緯度來動態計算 Haversine 距離，並重新動態排序、選出最近的 TOP 3 站點進行展示。
   * **可借車分色**：預設為深藍色 (`#0000AA`)，若小於等於 2 輛則變為紅色 (`#CC0000`) 警示。
   * **可還車分色**：預設為深綠色 (`#00AA00`)，若小於等於 2 輛則變為紅色 (`#CC0000`) 警示。

4. **公車即時到站**：
   * 捨棄緊密連線的 `<table>`，改用 Flexbox 獨立卡片清單。每條路線均為獨立且帶有白色底色、直角邊框的卡片，卡片間距露出網頁背景底色。
   * **方向分色與緊急高亮**：「去程」資料整行文字顯示為深藍色 (`#0000AA`)；「返程」資料整行文字顯示為深綠色 (`#00AA00`)。若到站時間少於或等於 2 分鐘（含「即將到站」），該欄位會轉為紅色 (`#CC0000`) 加粗字體。
   * **顯示上限控制**：排序後僅展示最快到站的前 25 筆公車路線，避免大型轉運站（如南港車站）元件過多排版崩潰。

5. **🚗 即時路況監控**：
   * **地理圍欄限制**：若中心定位點非「台北市」，系統將會以黃色警告框提示不支援。
   * **週邊過濾與半徑擴充**：定位在台北市區內時，系統會篩選出最接近當前定位點的 6 個主要觀測點路況。若 5 公里內無觀測點，會自動將搜尋半徑擴大至 10 公里。
   * **雙向車速與路況事件**：
     * 左側「週邊道路車流速限」：明確標示出各觀測點的行車方向（如：東向、西向、南向、北向）與對應方向之車速，並標註紅（壅塞）、黃（車多）、綠（順暢）三色。
     * 右側「即時路況突發事件」：若週邊路段發生突發事件，會以紅/黃邊框的警示卡片呈現詳細的文字描述，並在標題右側以小字顯示事件發生的時間。
   * **時間步長快取**：以 5 分鐘為基準模擬 Mock 資料，在 API 降級時能維持資料狀態一致性。

6. **周邊地圖 (定位校正)**：
   * 採用最穩定的 Google Maps 簡化版 Iframe 嵌入，保證 100% 成功加載且不受 API Key 權限限制。
   * **地圖定位校正**：已將「統一數位大樓 (內湖石潭路155號)」的預設座標精準校正為 **`(25.063549, 121.589583)`**，紅針精確指向正確的大樓地址門牌。

7. **美食推薦 (午餐推薦)**：
   * **動態搜尋與 Fallback**：若配置了 Google Places API Key，會以搜尋端點動態抓取周邊 1500 公尺內的 20 家熱門餐廳，並展示 `st.spinner("正在尋找附近最新美食推薦...")` 載入動畫。無 Places 金鑰或 API 失敗時，會自動 fallback 至本地 SQLite 資料庫。
   * **白色底色**：為整個摺疊面板（`st.expander`）統一套用強制的 `#FFFFFF` 白色背景，提升視覺對比度與易讀性。

8. **常用地標定位與行政區快取連動**：
   * 常用地標下拉選單中包含 **`南港新富公園`**（富康公園）、**`統一數位大樓 (內湖石潭路155號)`** 等。
   * 內建預置座標與行政區快速比對，自動判定 `台北市南港區` 與 `台北市內湖區`，避開 API 呼叫限制。

9. **便捷控制與導覽**：
   * **手動更新按鈕**：右上角點擊重刷（採用 Base64 編碼的 `UPDATE.png` 圖示），強制清除 API 快取，回傳 `?refresh=1` 後自動清除參數並重載，避免陷入循環重刷。
   * **回到最上方按鈕**：右下角固定定位 `BackToTop_2.png` 浮動按鈕，懸停放大 10% 且觸發 Newsprint 黑白反色濾鏡，點選平滑捲動回頂部。

### 📌 後台管理系統 (CRUD)
1. 展開左側側邊欄，勾選「**切換至後台管理**」。
2. 輸入管理員帳號：`admin`、密碼：`admin` 登入。
3. 登入成功後，即可使用視覺化可編輯表格對 SQLite 資料庫中的 **公告資料表** 與 **餐廳資料表** 進行新增、修改與刪除 (CRUD) 操作，儲存後即時生效於主頁。
